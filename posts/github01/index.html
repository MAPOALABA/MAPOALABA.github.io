Github.io와 Hugo를 이용한 블로그 구축#

우선 나는 Mac OS를 사용중이기 때문에 Mac을 기준으로 설명할 예정이다. 설명하기에 앞서 수많은 가이드 영상과 공식문서, 블로그를 접하며 시도했음에도 수많은 오류를 접해온 경험을 바탕으로 설명할 예정이다.

1. Hugo 설치#

  1. 터미널을 실행

  2. 터미널에 아래의 명령어를 입력

$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Homebrew를 설치하는 과정이다.

$ brew install hugo

Homebrew를 이용해 Hugo를 설치하는 과정이다.

  1. 터미널에서 아래 명령어를 통해 설치가 잘 되었는지 확인
$ hugo version

설치가 잘 되었다면 hugo v0.113.0+extended darwin/arm64 BuildDate=unknown 이런식으로 나올 것이다.

2. Git 설치#

  1. 터미널에서 아래의 명령어 입력
$ brew install git

Git을 설치하는 과정

  1. 터미널에서 아래 명령어를 통해 설치가 잘 되었는지 확인
git version

설치가 잘 되었다면 git version 2.40.0 이라고 출력될 것이다.

3. Github 저장소 2개 생성#

  • 하나는 Hugo의 컨텐츠와 소스파일을 포함할 <프로젝트 이름> 저장소 생성(나는 blog로 생성함)

  • 다른 하나는 렌더링된 버전의 Hugo 웹사이트를 포함할 <Username>.github.io 저장소 생성(나의 경우에는 mapoalaba.github.io)

4. Hugo 프로젝트 만들기#

터미널에서 될 수 있다면 한글이 없는 경로로 접속한 후

$ hugo new site <프로젝트 이름> -f -yml

이제 프로젝트 이름으로 폴더가 하나 생성될 것이다. 폴더에 들어가보면 기본적인 구조가 잡혀있을 것이다.

5. 사이트 테마#

$ git clone <테마 github 저장소 주소>

터미널에서 $ cd 프로젝트 폴더로 들어가 위 명령어를 작성해준다.

이후 hugo.yml 파일에 들어가서 본인이 고른 hugo 테마에 들어가서 hugo.yml 견본파일을 참고하며 파일 내용을 수정해주도록 하자.

프로젝트 폴더에 들어갔을 때 hugo.toml 파일이 있다면, hugo.yml로 이름을 바꿔주도록 하자

※주의※ 프로젝트 폴더안에 있는 themes 폴더의 내용은 절대 건들지 말자.

블로그가 생성은 되었겠지만, 이젠 디자인을 해야한다. 우선 hugo.yml 파일에 들어가서

6. 게시물 생성#

/프로젝트 폴더/content/posts 경로로 들어가서 <파일이름>.md MarkDown 형식의 파일을 생성해준다. 생성 후에는

---
title: "블로그 구축 가이드"
date: 2023-04-04T11:30:03+00:00
# weight: 1
# aliases: ["/first"]
tags: ["GitHub"]
categories: ["GitHub"]
author: "Me"
# author: ["Me", "You"] # multiple authors
showToc: true
TocOpen: false
draft: false
hidemeta: false
comments: false
description: "Desc Text."
canonicalURL: "https://canonical.url/to/page"
disableHLJS: true # to disable highlightjs
disableShare: false
disableHLJS: false
hideSummary: false
searchHidden: true
ShowReadingTime: true
ShowBreadCrumbs: true
ShowPostNavLinks: true
ShowCodeCopyButtons: true
ShowWordCount: true
ShowRssButtonInSectionTermList: true
UseHugoToc: true
cover:
    image: cover/blog.jpg
    alt: "<alt text>" # alt text
    caption: "<text>" # display caption under cover
    relative: false # when using page bundles set this to true
    hidden: false # only hide on current single page
editPost:
    URL: "https://github.com/<path_to_repo>/content"
    Text: "Suggest Changes" # edit text
    appendFilePath: true # to append file path to Edit link
---

이 코드를 그대로 붙여넣고 수정해야할 부분은 title, date, tags, categories, cover 이다. 이후 --- 아래에는 MarkDown 문법으로 게시글을 작성하면 된다. MarkDown 문법은 구글링하면 아주 쉽게 정리되어 있다.

7. 사이트 Git Push#

여기서 주의 할 점은 프로젝트 폴더에 Public 폴더가 존재해 있으면 안된다. 만약 있다면 지우고 아래의 명령어를 터미널에 입력하자.

$ git init
$ git remote add origin <첫 번째 저장소 URL>

이제 public 폴더를 두번째 저장소와 연결할 것이다.

$ git submodule add -b master <두번째 저장소 URL> public

이제 프로젝트 폴더에서

$ hugo -t <테마 이름>

나같은 경우는 PaperMod 라는 테마를 사용했기에

hugo -t PaperMod 입력함

이제 public 폴더에 많은 파일들이 생성됐을 것이다. 이제 다 끝났다 저장소에 파일을 push만 하면 된다. 프로젝트 폴더로 들어간 후

$ cd public
$ git add .
$ git commit -m "<커밋 메세지>"
$ git push origin master
$ cd ..
$ git add .
$ git commit -m "<커밋 메세지>"
$ git push origin master

이제 블로그가 생성되었다.

<Username>.github.io 를 주소창에 입력하면 만든 블로그에 들어갈 수 있다.

8. 앞으로의 Push 방법#

필자의 경우는 두번째 Push할때부터 많은 오류를 접했다. 따라서 아래의 명령어를 순서도 틀리지말고 그대로 따라하는걸 추천한다.

$ cd 프로젝트 폴더
$ hugo -t 테마 이름
$ cd public
$ git add .     //add 띄고 .이다
$ git commit -m "커밋 메세지"
$ git push origin master
$ cd ..
$ git add .
$ git commit -m "커밋 메세지"
$ git push origin master

최대한 필자가 겪은 어려움을 바탕으로 작성했다. 구글링하면 다 나오기에 이 글을 보고도 오류가 난다면 수많은 시행착오를 스스로 겪으며 극복하길 바란다.
