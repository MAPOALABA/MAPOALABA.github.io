운영체제란 ?#

  • Operating System 이라고 일컫는 운영체제는 편리한 인터페이스 환경을 제공하고 컴퓨터 시스템의 자원을 효율적으로 관리하는 일종의 소프트웨어이다.

자원 : CPU, RAM, HDD, File System

운영체제의 역할과 목표#

  • 운영체제의 역할

    1. 자원 관리 : 여러 응용 프로그램이 자원을 요청하면 적절한 적절한 순서로 배분하고 회수하여 자원을 효율적으로 관리 => 효율성

    2. 자원 보호 : CPU, 메모리 등에 대한 사용자와 응용 프로그램의 직접 접근을 막음=> 안정성

    3. 하드웨어 인터페이스 제공 : 마우스, 키보드 등을 사용할 수 있도록 하드웨어 인터페이스를 제공한다.=> 확장성

    4. 소프트웨어 인터페이스 제공 : 사용자가 운영체제를 편리하게 사용하기 위해 제공되는 것=> 편리성

자원관리, 자원보호, 하드웨어/소프트웨어 인터페이스 제공을 하는 운영체제는 효율성, 안정성, 확장성, 편리성을 목표로 함

운영체제의 구조#

운영체제의 목적 (출제)#

  1. 사용자의 편리성

  2. 성능

    • 처리능력 : 주어진 시간 내에 작업 처리 능력

    • 신뢰성(정확성) : 얼마나 정확한가?

    • 응답 시간 : 반응 속도

    • 가용성 : 언제든지, 멈추지 않고

    메쉬형이 가용성 Best

운영체제의 내부 기술#

  1. 일괄처리 시스템(배치 처리 방식)
    • 일정량/기간 동안의 데이터를 모아서 한꺼번에 처리

유휴 시간 발생

  1. 다중 프로그래밍 시스템

    • 하나의 CPU와 주기억장치를 이용하여 여러 개의 프로그램을 동시에 처리하는 방식

  2. 시분할 시스템(RR)

    • 여러 명의 사용자가 사용하는 시스템에서 컴퓨터가 사용자들의 프로그램을 번갈아 가며 처리해 줌으로써 각 사용자에게 독립된 컴퓨터를 사용하는 느낌을 주는 것

라운드 로빈 방식

  1. 실시간 처리 시스템

    • 데이터 발생 즉시, 또는 데이터 처리 요구가 있는 즉시 처리하여 결과를 산출하는 방식

  2. 분산 처리 시스템

    • 여러 개의 컴퓨터(프로세스)를 통신 회선으로 연결하여 하나의 작업을 처리하는 방식

  3. 다중 처리 시스템

    • 여러 개의 CPU와 하나의 주기억장치를 이용하여 여러 개의 프로그램을 동시에 처리하는 방식

하나의 CPU가 고장나도 다른 CPU를 사용하면 되기에 신뢰성, 안정성이 높음

  1. 병렬 처리 시스템
    • 한 대의 컴퓨터에 여러 개의 CPU를 이용하여 여러 작업을 동시다발적으로 처리하는 시스템

계층형 구조 커널#

계층형 구조 커널 layered architecture은 단일형 구조 커널의 발전된 형태로 비슷한 기능의 모듈을 하나의 계층으로 묶어 계층 간의 통신을 통해 운영체제를 구현함

  • 계층형 구조 커널은 버그나 오류가 생길 경우 해당 계층만 수정하면 되기 때문에 디버깅이 쉽다는 장점이 있음

정리#

운영체제는 사용자에게 편리한 인터페이스 환경을 제공하고 컴퓨터 시스템의 자원을 효율적으로 관리한다.
커널과 인터페이스로 구성되어 있으며,
커널은 system call과 driver를 통해
응용 프로그램, 하드웨어가 자원에 접근할 수 있도록 한다.
